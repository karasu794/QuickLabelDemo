# QuickLabel 送り状作成ページ レスポンシブ対応実装

## 概要
QuickLabel の送り状作成ページを、Tailwind CSS のモバイルファーストアプローチを使用して、スマートフォンからデスクトップまで最適な体験を提供できるよう完全にレスポンシブ対応させました。

## 実装方針

### モバイルファーストアプローチ
- **基本スタイル**: モバイル向けのデザインを基準として実装
- **ブレークポイント**: `md:`（768px以上）でデスクトップ向けレイアウトに切り替え
- **タッチフレンドリー**: モバイルでのタップ操作を考慮したUI要素のサイズ調整

### デザイン哲学
- **情報密度の最適化**: 画面サイズに応じて適切な情報密度を提供
- **直感的なナビゲーション**: ユーザーの現在位置と次のアクションが明確
- **一貫性**: デバイス間でブランド体験とユーザビリティを統一

## 主要な変更点

### 1. ステップナビゲーションの適応型レイアウト

#### 🖥️ デスクトップ（md以上）
```tsx
<div className="hidden md:block md:col-span-3">
  <div className="bg-white rounded-lg shadow-sm p-4 lg:p-6 sticky top-6">
    {/* 従来の詳細ステップナビゲーション */}
  </div>
</div>
```

#### 📱 モバイル（md未満）
```tsx
<MobileStepProgress />
```

**特徴:**
- **コンパクトなプログレスバー**: 水平プログレスバーと現在ステップの表示
- **前後ナビゲーション**: 直感的な前後移動ボタン
- **ステップ概要**: 横スクロール可能なステップ名一覧

### 2. レイアウト構造の適応

#### デスクトップレイアウト
```
┌─────────────────────────────┬─────────────┐
│                             │   ステップ   │
│       メインコンテンツ       │  ナビゲーション │
│       （9列分）             │   （3列分）   │
│                             │             │
└─────────────────────────────┴─────────────┘
```

#### モバイルレイアウト
```
┌─────────────────────────────────────────────┐
│           コンパクトプログレスバー            │
├─────────────────────────────────────────────┤
│                                             │
│          メインコンテンツ（フル幅）           │
│                                             │
└─────────────────────────────────────────────┘
```

### 3. フォーム要素の最適化

#### 入力フィールド
```tsx
// モバイル: より高いタップターゲット
className="h-11 md:h-10 text-base"

// デスクトップ: 標準的な高さ
```

#### ラベルとテキスト
```tsx
// フォントサイズの調整
className="text-sm md:text-base"

// 見出しサイズの調整  
className="text-xl md:text-2xl"
```

#### レイアウトグリッド
```tsx
// モバイル: 縦一列、デスクトップ: 横並び
className="grid grid-cols-1 md:grid-cols-2 gap-4 md:gap-6"
```

#### ボタンレイアウト
```tsx
// モバイル: 縦スタック、デスクトップ: 横並び
className="flex flex-col sm:flex-row justify-between gap-3 sm:gap-4"

// モバイルでの重要性順序（次へボタンを上に）
className="order-1 sm:order-2" // 次へボタン
className="order-2 sm:order-1" // 戻るボタン
```

## コンポーネント別実装詳細

### MobileStepProgress.tsx
**新規作成されたモバイル専用コンポーネント**

#### 主要機能
1. **現在ステップの表示**: 大きなアイコンとタイトルで現在位置を明示
2. **プログレスバー**: 全体の進捗を視覚的に表示
3. **ステップドット**: 各ステップの完了状態を色分けで表示
4. **ナビゲーション**: 前のステップへの戻りボタン
5. **ステップ一覧**: 横スクロール可能な全ステップ概要

#### デザイン特徴
```tsx
// 進捗に応じた色分け
- アクティブ: purple-600 (紫)
- 完了: orange-400 (オレンジ) 
- 未完了: gray-300 (グレー)

// コンパクトなレイアウト
- パディング: p-4 py-3
- 高さ制限で情報密度を最適化
```

### レイアウトファイル (layout.tsx)
#### レスポンシブグリッド実装

## ブレークポイント戦略

### Tailwind CSS ブレークポイント使用
- `sm:` 640px以上 - 小さなタブレット向け微調整
- `md:` 768px以上 - タブレット・デスクトップ向けメインレイアウト
- `lg:` 1024px以上 - 大画面デスクトップ向け追加調整

### 段階的拡張
```scss
/* モバイル（基本スタイル） */
.element { /* モバイル向けスタイル */ }

/* タブレット以上 */
@media (min-width: 768px) {
  .md\:element { /* タブレット向け調整 */ }
}

/* デスクトップ */
@media (min-width: 1024px) {
  .lg\:element { /* デスクトップ向け追加調整 */ }
}
```
